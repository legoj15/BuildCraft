## 2024-05-19 - Fastutil primitive collections for BlockPos **Learning:** `BlockPos` sets can grow extremely large in pathfinding or flood-fill algorithms (like Pump scanning), causing massive object allocation via `HashSet<BlockPos>`. Using `it.unimi.dsi.fastutil.longs.LongOpenHashSet` with `BlockPos.asLong()` drastically reduces this GC overhead and is natively supported via NeoForge/Minecraft dependencies. **Action:** Prefer primitive `LongOpenHashSet` over `HashSet<BlockPos>` for high-volume coordinate tracking.
## 2024-05-27 - Entity Lookup Optimization **Learning:** In Minecraft NeoForge modding, `Level#getEntities` without a specific class filter scans and allocates all entities in the area, which is highly inefficient. **Action:** Always use `Level#getEntitiesOfClass(SpecificEntity.class, ...)` when looking for a specific type of entity to leverage internal spatial/type tracking and reduce list allocations.
