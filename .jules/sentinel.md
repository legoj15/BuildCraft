## 2024-05-18 - [Missing Distance Verification on Debug Requests] **Vulnerability:** Server trusting client position on C2S packet. **Learning:** C2S packets like `MessageDebugRequest` lacked distance verification. **Prevention:** Always check `distanceToSqr` for `BlockEntity` coordinate interactions.
## 2026-04-10 - [Missing Container Validity Check on Payloads] **Vulnerability:** Server accepting arbitrary C2S container payloads without checking if the container is still valid for the player. **Learning:** C2S packets like `MessageContainerPayload` can be used to interact with block entities from anywhere if distance/validity checks are skipped. **Prevention:** Always call `bcContainer.stillValid(player)` on the server before processing container payloads.
## 2026-04-12 - [Container Message Reach Exploit] **Vulnerability:** Missing distance validation in C2S container payload allowed out-of-range reach/interact exploits. **Learning:** While the server typically closes an `openContainer` when a player moves out of range, there's a race condition between block breaking/moving and the server ticking the player to close the container. Custom C2S payloads routing through containers do not automatically re-verify range. **Prevention:** Always call `container.stillValid(player)` server-side when processing custom payloads related to open containers.
## 2024-05-18 - [Missing Distance Validation in Container Payloads] **Vulnerability:** Server accepting client widget interactions regardless of player distance. **Learning:** Custom C2S payloads like `MessageContainerPayload` must explicitly verify `bcContainer.stillValid(player)` because vanilla UI safety checks only apply to standard container interactions. **Prevention:** Always call `stillValid(player)` server-side when receiving container-scoped network payloads.
