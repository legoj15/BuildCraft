Fix Gate statement persistence: legacy NBT converter was destroying saved data

The 'Legacy' NBT conversion code in GateLogic.readConfigData() was
unconditionally running on every load, extracting 'kind' and 'side'
from the root of each trigger/action compound tag. However,
FullStatement.writeToNbt() stores statement data INSIDE a nested
's' sub-compound: { "s": { "kind": "...", "side": 6 } }

The legacy converter looked for 'kind' at the root level, found nothing
(empty string), and replaced the entire tag with { "kind": "", "side": 6 }.
This erased the properly-saved 's' compound on every load, making all
configured gate statements silently vanish on server restart/rejoin.

Fix: Only apply the legacy conversion when 'kind' exists at root level
AND the 's' sub-compound is absent (true 1.12.2 format). When the
modern 's' format is present, skip the conversion entirely.

Also removed diagnostic [GateSync] logging and cleaned up stray
commit_msg2.txt.
